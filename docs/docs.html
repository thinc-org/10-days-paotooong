Authenication

Once user is register and login, user can authorized with Authorization header with bearer token

For example:

curl -X POST -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...." /v1/wallet/topup

If you get Unauthorized, this mean that you have not put in your token in the headeror your token is invalid provided (expired, incorrect hash).

The token is configured to expired in 10 days, hard-coding it is also a choice.
